# Kondo — Visão Geral da Plataforma

> **"Kondo transforma a vida em condomínio de manual e opaca em uma operação transparente, automatizada e preparada para crédito."**

---

## O que é o Kondo

Kondo é uma plataforma **AI-native para gestão condominial**. Ela conecta síndico, conselho e morador em fluxos operacionais rastreáveis, transparentes e apoiados por inteligência artificial.

Na prática, o Kondo funciona como uma administradora digital: reduz tarefas manuais do síndico, organiza a governança do condomínio e dá ao morador um canal único para resolver problemas e acompanhar status.

---

## O problema que resolve

Condomínios ainda dependem de planilhas, PDFs, e-mails e mensagens de WhatsApp para tudo — chamados, prestação de contas, inadimplência, documentos e comunicados.

Para cada persona, isso gera uma dor distinta:

| Persona | Dor principal |
|---|---|
| **Síndico** | Chamados se perdem em canais soltos. Balancetes e inadimplência exigem retrabalho manual. Comunicados consomem tempo. |
| **Conselho** | Recebe PDFs densos sem contexto. Não consegue auditar decisões nem acompanhar o caixa em tempo real. |
| **Morador** | Abre chamado por mensagem e nunca sabe o status. Não tem canal único para regras, avisos e demandas. |

---

## As três experiências da plataforma

### Síndico — Controle e automação

O painel do síndico mostra o que precisa de ação hoje: chamados críticos, risco de caixa, contratos vencendo e inadimplência acumulada.

- Dashboard de prioridades com alertas em tempo real
- Chamados classificados automaticamente por IA (categoria, prioridade, responsável)
- Kanban operacional com rastreamento de status ponta a ponta
- Simulador de acordos de inadimplência com impacto no fluxo de caixa
- Geração de comunicados a partir de uma instrução simples
- Gestão de fornecedores, orçamentos e comparação por IA

### Conselho — Transparência e governança

O painel do conselho centraliza tudo que os membros precisam para auditar a gestão sem depender de reunião mensal.

- Visão consolidada de receitas, despesas, inadimplência e caixa projetado
- Histórico de decisões, obras e acordos com justificativas
- Acesso a documentos: regimento, atas, contratos, balancetes
- Resumo financeiro em linguagem simples gerado por IA
- Rastreamento de manutenções e obras em andamento

### Morador — Self-service e visibilidade

O portal do morador entrega o essencial sem burocracia.

- Abertura de chamados com acompanhamento de status em tempo real
- Consulta ao regimento por linguagem natural ("Posso fazer obra no sábado?")
- Comunicados e avisos centralizados
- Histórico de chamados da unidade
- Boletos e calendário de eventos do condomínio

## Kondo AI — O copiloto da plataforma

O **Kondo AI** é o clímax do sistema. É o ponto onde todos os módulos, dados e permissões convergem em uma única interface conversacional.

Em vez de navegar por telas para executar tarefas, o usuário fala com o Kondo AI em linguagem natural — e o sistema entende, consulta os dados certos e age.

### O que torna o Kondo AI diferente

A maioria dos chatbots em produtos SaaS responde perguntas. O Kondo AI **age sobre o sistema**.

Dependendo do perfil do usuário, o Kondo AI pode:

- Abrir chamados operacionais diretamente no Kanban
- Registrar despesas no financeiro
- Criar e publicar comunicados para moradores
- Atualizar o status de chamados em andamento
- Responder perguntas sobre documentos do condomínio
- Resumir o estado financeiro em linguagem simples
- Sugerir próximos passos baseados nos dados reais do condomínio

Cada ação é auditada, rastreada e associada ao usuário que a executou — exatamente como se fosse feita manualmente nas telas.

### Capacidades por perfil

O Kondo AI opera com **permissões espelhadas ao perfil do usuário**. Não existe uma IA com acesso irrestrito — cada conversa respeita exatamente os mesmos limites que a interface gráfica.

#### Síndico

O síndico tem acesso completo. O Kondo AI pode:

| Ação | Exemplo de comando |
|---|---|
| Abrir chamado | "Abre chamado de vazamento no corredor bloco B, unidade 304" |
| Registrar despesa | "Registra despesa de manutenção de portão, R$ 1.800, categoria manutenção, vencimento 20/07" |
| Criar comunicado | "Cria comunicado informando que a piscina estará fechada sexta por manutenção" |
| Publicar comunicado | "Publica o comunicado para os moradores" |
| Atualizar status de chamado | "Muda o chamado 42 para fornecedor acionado" |
| Consultar financeiro | "Qual é o caixa atual? Tem risco de fechar negativo esse mês?" |
| Listar prioridades | "O que precisa de ação hoje?" |

Para operações de alto impacto — despesas acima de R$ 5.000, publicação imediata de comunicados e resolução de chamados — o Kondo AI exige confirmação explícita antes de executar. Nenhuma alteração é gravada sem que o usuário responda com uma confirmação clara.

#### Conselho

O conselho tem acesso de governança. O Kondo AI pode:

| Ação | Exemplo de comando |
|---|---|
| Consultar financeiro | "Qual a inadimplência atual e o impacto no caixa?" |
| Abrir chamado | "Abre chamado sobre iluminação quebrada no estacionamento" |
| Criar comunicado | "Prepara comunicado sobre a assembleia de quarta" |
| Resumir estado do condomínio | "Me dá um resumo para a reunião do conselho de hoje" |

O conselho **não pode** registrar despesas nem alterar status de chamados. Qualquer tentativa é bloqueada com explicação clara.

#### Morador

O morador tem acesso focado em self-service. O Kondo AI pode:

| Ação | Exemplo de comando |
|---|---|
| Abrir chamado | "Tem um vazamento no banheiro da minha suíte" |
| Consultar regras | "Posso fazer barulho até que horas?" |
| Acompanhar chamados | "Qual é o status do meu chamado de infiltração?" |

O morador **não tem acesso** a dados financeiros do condomínio, não pode criar comunicados e não pode alterar status de chamados. O Kondo AI não contorna essas restrições — explica o limite e oferece o que pode fazer dentro do perfil.

### Guardrails de segurança

O Kondo AI possui múltiplas camadas de proteção que operam antes e durante cada conversa:

1. **Preflight por perfil** — antes de processar a mensagem, o sistema verifica se a intenção é compatível com o perfil. Uma tentativa de morador acessar financeiro é bloqueada antes de chegar ao modelo de IA.

2. **Confirmação obrigatória** — operações de alto impacto (despesas altas, publicação imediata, encerramento de chamados) exigem resposta explícita do usuário antes de serem gravadas.

3. **Auditoria de todas as ações** — cada operação executada pelo Kondo AI gera um `AuditEvent` com o usuário, a ação, a entidade afetada e os dados antes/depois.

4. **Sem decisões sensíveis autônomas** — o Kondo AI não pode perdoar dívidas, aprovar gastos sem confirmação, iniciar cobrança jurídica ou tomar qualquer decisão que exija julgamento humano.

### Sessões persistidas

O Kondo AI mantém histórico de conversa por sessão. O contexto não se perde entre mensagens — o modelo lembra o que foi dito antes e pode retomar fluxos interrompidos.

Cada sessão é vinculada ao usuário e ao condomínio, garantindo que históricos não vazem entre usuários ou condomínios diferentes.

### Fallback sem chave de IA

O sistema funciona mesmo sem uma chave de API de IA configurada. Nesse modo, o Kondo AI usa respostas baseadas nos dados reais do condomínio (chamados, financeiro, documentos) para responder perguntas básicas e orientar o usuário — sem inventar informações.

Quando a chave está configurada, o modelo usa **Claude** (Anthropic) como motor de raciocínio, com suporte a envio de imagens e documentos junto com a mensagem.

### Onde acessar

O Kondo AI fica em **`/kondo-ia`** — uma tela dedicada acessível a todos os perfis pela navegação lateral. Além disso, um painel de copiloto contextual (`CopilotPanel`) está disponível em páginas-chave do sistema, permitindo perguntas rápidas sem sair da tela atual.

## Fluxo de referência: chamado do morador ao síndico

O fluxo abaixo ilustra como os módulos e o Kondo AI se integram em um cenário real.

```
Morador abre conversa com Kondo AI
  → Descreve: "Tem água escorrendo do teto do meu banheiro"
  → Kondo AI abre chamado direto no sistema (título, descrição, localização)
  → WorkItem criado automaticamente no Kanban
  → Síndico vê chamado classificado no dashboard (categoria: hidráulica, prioridade: alta)
  → Síndico usa Kondo AI: "Muda chamado 47 para fornecedor acionado"
  → Kondo AI atualiza status e sincroniza Kanban
  → Síndico cria comunicado: "Informa que área B está em manutenção hidráulica"
  → Comunicado publicado e visível para moradores
  → Morador acompanha status pelo portal sem precisar ligar para o síndico
```

## Como rodar

### Docker (recomendado)

```bash
cd kondo
docker compose up --build
```

Sobe API em `http://localhost:8000` e PostgreSQL em `localhost:5432`. Banco populado automaticamente com dados demo.

### Local (SQLite)

```bash
# Backend
cd kondo
cp .env.example .env
uv sync
uv run python -m app.seed
uv run uvicorn app.main:app --reload

# Frontend (outro terminal)
cd kondo-front
bun install
bun run dev
```

Frontend em `http://localhost:5173` · API em `http://localhost:8000` · Swagger em `http://localhost:8000/docs`

### Ativar o Kondo AI com modelo real

No arquivo `.env` do backend:

```env
ANTHROPIC_API_KEY=sk-ant-...
AI_PROVIDER=anthropic
AI_MODEL=claude-haiku-4-5
```

Sem essa configuração, o sistema usa fallback simulado baseado nos dados reais do banco.
